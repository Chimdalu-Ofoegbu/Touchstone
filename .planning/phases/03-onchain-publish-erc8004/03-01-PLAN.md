## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external caller → publishRating | Untrusted msg.sender attempts to write a rating; only the ERC-8004 NFT holder may pass |
| RatingRegistry → ERC-8004 registry | Cross-contract `ownerOf` call to an external contract (the new failure surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | publishRating gate | mitigate | Live `ownerOf(agentTokenId) == msg.sender` gate; `test_publishRating_revertsForNonAgent` (written FIRST, D-01) proves the revert. The load-bearing control. |
| T-03-02 | Tampering | gate target network | mitigate | Registry address is an immutable constructor arg + a named `IDENTITY_REGISTRY` constant in Deploy.s.sol with a "Mantle Mainnet ONLY" comment — no Sepolia address can be silently substituted. |
| T-03-03 | Tampering / Repudiation | hash without retrievable reasoning | mitigate | `cid` is written in the SAME `publishRating` call as `reasoningHash` (D-02) — the struct can never hold a hash with no pointer. |
| T-03-04 | Information Disclosure | cross-contract return-value trust | accept | `ownerOf` returns a single address compared by `==`; no return-value-length or reentrancy surface (view call, no state change after). Low risk — `view` external call cannot reenter and mutate. |
</threat_model>
