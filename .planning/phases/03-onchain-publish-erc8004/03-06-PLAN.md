## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chain event → watcher | Untrusted RatingRequested(subject) from any caller; subject address must be allow-listed before rating |
| watcher → publish pipeline | The daemon drives signed Mainnet writes; double-fire must be harmless |
| continuous log stream | The daemon logs forever; must never leak the keyed RPC URL or any secret |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-20 | Tampering | watcher + manual CLI double-publish race | mitigate | `inFlight` Set dedupe (D-03); history is append-only so a duplicate publish is harmless; test proves single-invoke under double-fire. |
| T-03-21 | Spoofing | requestRating for an arbitrary/unknown address | mitigate | `subjectIdFromAddress` returns null for unknowns → watcher logs+skips, never reaches rate()/Claude (Pitfall 6 + allow-list discipline). |
| T-03-22 | Denial of Service | silent subscription drop stops all publishes mid-demo | mitigate | `onError` → unwatch + reconnect with backoff (Pitfall 3); heartbeat makes liveness visible; manual `pnpm publish-rating` is the rehearsed fallback (D-03). |
| T-03-23 | Information Disclosure | keyed RPC URL / secret in the continuous log stream | mitigate | Every caught error funnels through redactRpcError; the daemon never logs PRIVATE_KEY/PINATA_JWT. |
| T-03-24 | Denial of Service | out-of-gas mid-demo | mitigate | Pre-flight `cast balance` check (D-03); agent funded 16.41 MNT. |
